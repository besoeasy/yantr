Install Docker

Everything you need to install and run Docker before installing Yantr.

What is Docker?

Docker packages software into containers — isolated environments that include everything an app needs to run: code, runtime, libraries, and configuration.

A container behaves the same on every machine: it doesn't depend on the host OS or other installed software.

Why Yantr needs Docker

Isolation

Each app runs in its own container. They can't interfere with each other or with your system.

No conflicts

Apps that need different versions of the same library coexist without issues. No more dependency hell.

Clean removal

Stop a container and it's gone. No leftover files, services, or registry entries on your system.

Installation guide

Requirements

Windows 11 with WSL 2 enabled (recommended) or Hyper-V. 64-bit; 4 GB RAM minimum.

Step 1 — Enable WSL 2

wsl --install

Run this command in an elevated PowerShell (Run as Administrator), then restart when prompted.

Step 2 — Install Docker Desktop

Download and run the installer for your CPU architecture from the official Docker website:

docker.com/desktop → Windows

Step 3 — Verify

docker run hello-world

Requirements

macOS 12 (Monterey) or newer. Works on both Intel and Apple Silicon (M1/M2/M3).

Step 1 — Install Docker Desktop

Download the correct installer for your chip from the official Docker website:

docker.com/desktop → Mac

Or install with Homebrew

brew install --cask docker

Step 2 — Verify

docker run hello-world

Requirements

Ubuntu, Debian, Fedora, or similar (64-bit). The script below works on most distributions.

Step 1 — Install Docker Engine

curl -fsSL https://get.docker.com | sudo sh

Step 2 — Run Docker without sudo

sudo usermod -aG docker $USER && newgrp docker

Log out and back in (or restart your session) if the group change doesn't take effect immediately.

Step 3 — Verify

docker run hello-world

Requirements

Any 64-bit Linux server (Ubuntu Server, Debian, CentOS, RHEL, or similar). SSH access required.

Step 1 — Install Docker Engine

curl -fsSL https://get.docker.com | sudo sh

Step 2 — Run Docker without sudo

sudo usermod -aG docker $USER && newgrp docker

Step 3 — Enable Docker on boot

sudo systemctl enable --now docker

This ensures Docker and your containers restart automatically after a reboot.

Step 4 — Verify

docker run hello-world

Requirements

Raspberry Pi 3 or newer, running Raspberry Pi OS (64-bit recommended) or Ubuntu for Raspberry Pi. 2 GB RAM minimum.

Raspberry Pi uses ARM architecture. Most popular Docker images support ARM64 — check the image page on Docker Hub if an app fails to start.
